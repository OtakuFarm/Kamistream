KamiStream needs a real database so users, videos, and votes persist across devices.

Create a free Supabase account & project
✓

Go to supabase.com → click Start your project → sign in with GitHub or email → click New project → give it a name like kamistream → choose a region close to you → set a database password → click Create new project.

Wait ~2 minutes for the project to spin up, then continue.

Run the database schema
✓

In your Supabase dashboard → click SQL Editor in the left sidebar → click New query → paste the SQL below → click Run.

-- KamiStream Schema — paste into Supabase SQL Editor and click Run -- Safe to run multiple times (uses IF NOT EXISTS) create table if not exists public.profiles ( id uuid references auth.users on delete cascade primary key, username text unique not null, level integer default 1, created_at timestamptz default now() ); create table if not exists public.weeks ( id serial primary key, week_key text unique not null, theme text default 'Best Fight Scene Reaction', prize integer default 20, starts_at timestamptz not null, ends_at timestamptz not null, is_active boolean default true, created_at timestamptz default now() ); create table if not exists public.videos ( id text primary key default gen_random_uuid()::text, user_id uuid references public.profiles on delete cascade not null, week_id integer references public.weeks not null, username text not null, caption text, video_url text not null, embed_url text, platform text, thumbnail_url text, video_id text, likes integer default 0, views integer default 0, is_removed boolean default false, submitted_at timestamptz default now() ); create table if not exists public.likes ( id serial primary key, user_id uuid references public.profiles on delete cascade not null, video_id text references public.videos on delete cascade not null, created_at timestamptz default now(), unique(user_id, video_id) ); create table if not exists public.comments ( id serial primary key, user_id uuid references public.profiles on delete cascade not null, video_id text references public.videos on delete cascade not null, username text not null, content text not null, created_at timestamptz default now() ); create table if not exists public.winners ( id serial primary key, video_id text not null, user_id uuid not null, username text not null, week_id integer references public.weeks, prize integer default 20, paid_via text, created_at timestamptz default now() ); -- Enable RLS alter table public.profiles enable row level security; alter table public.videos enable row level security; alter table public.likes enable row level security; alter table public.comments enable row level security; alter table public.weeks enable row level security; alter table public.winners enable row level security; -- Drop existing policies first to avoid conflicts do $$ begin drop policy if exists "Public profiles" on public.profiles; drop policy if exists "Own profile" on public.profiles; drop policy if exists "Update own profile" on public.profiles; drop policy if exists "Public videos" on public.videos; drop policy if exists "Insert own video" on public.videos; drop policy if exists "Public likes" on public.likes; drop policy if exists "Like videos" on public.likes; drop policy if exists "Unlike videos" on public.likes; drop policy if exists "Public comments" on public.comments; drop policy if exists "Post comments" on public.comments; drop policy if exists "Public weeks" on public.weeks; drop policy if exists "Public winners" on public.winners; exception when others then null; end $$; -- Recreate policies create policy "Public profiles" on public.profiles for select using (true); create policy "Own profile" on public.profiles for insert with check (auth.uid() = id); create policy "Update own profile" on public.profiles for update using (auth.uid() = id); create policy "Public videos" on public.videos for select using (true); create policy "Insert own video" on public.videos for insert with check (auth.uid() = user_id); create policy "Update own video" on public.videos for update using (auth.uid() = user_id); create policy "Public likes" on public.likes for select using (true); create policy "Like videos" on public.likes for insert with check (auth.uid() = user_id); create policy "Unlike videos" on public.likes for delete using (auth.uid() = user_id); create policy "Public comments" on public.comments for select using (true); create policy "Post comments" on public.comments for insert with check (auth.uid() = user_id); create policy "Public weeks" on public.weeks for select using (true); create policy "Public winners" on public.winners for select using (true); -- Auto-create profile on signup create or replace function public.handle_new_user() returns trigger language plpgsql security definer as $$ begin insert into public.profiles (id, username) values ( new.id, coalesce(new.raw_user_meta_data->>'username', split_part(new.email, '@', 1)) ) on conflict (id) do nothing; return new; end; $$; drop trigger if exists on_auth_user_created on auth.users; create trigger on_auth_user_created after insert on auth.users for each row execute procedure public.handle_new_user(); -- Seed current week (safe, won't duplicate) insert into public.weeks (week_key, theme, prize, starts_at, ends_at) values ( 'week_' || to_char(date_trunc('week', now()), 'IYYY_IW'), 'Best Fight Scene Reaction', 20, date_trunc('week', now()), date_trunc('week', now()) + interval '7 days' ) on conflict (week_key) do nothing;

Information you provide:

  • Account information — email address and username when you register.

  • Contact messages — your name, email, and message if you contact us.

Information collected automatically:

  • Usage data — pages visited, features used, and navigation patterns.
  • Device information — browser type, OS, device type, and screen resolution.

  • Cookies and local storage — see Cookies section below.
We do not collect payment information, government ID, or any sensitive personal data. A real name is never required.
2. How We Use Your Information
  • To provide and operate the KamiStream platform — including your account, watchlist, and viewing history.
  • To run the weekly creator challenge — displaying submissions, counting votes, and contacting winners.
  • To improve our site — understanding how users navigate helps us build better features.
  • To serve relevant advertising — ad networks use cookie data to show you interest-based ads.
  • To send transactional emails — account confirmation and prize winner notifications only. No marketing emails without explicit consent.
  • To protect our platform — detecting abuse, spam, and Terms of Service violations.
3. Cookies & Tracking Technologies

Essential cookies — Required for the site to function. Includes session tokens and user preference settings (watchlist, continue-watching progress).

Analytics cookies — Used to understand how visitors use our site. Data is aggregated and anonymous. We may use services like Google Analytics.

Advertising cookies — Our ad network partners place cookies to track browsing behaviour and show relevant ads. This includes behavioural advertising via partners such as Google AdSense and other display or native ad networks.

Third-party embed cookies — Anime content is embedded from third-party video providers. Those providers may set their own cookies when you interact with their players. We do not control these cookies.

You can manage or disable cookies in your browser settings at any time. Disabling advertising cookies will not remove ads but may make them less relevant.

Advertising Networks — We display ads through third-party networks that may use cookies, web beacons, and similar technologies to collect data about your visits and provide targeted advertising.

Embedded Video Providers — Anime content is delivered via embedded players from third-party sources. We do not host, store, or distribute any video files. When you play a video, your browser connects directly to the third-party host's servers under their own privacy policy.

Authentication & Database — We use Supabase to manage user accounts and data. Supabase processes data on our behalf and adheres to industry-standard security practices.

Links on our site may lead to external websites. We are not responsible for the privacy practices of those sites.
5. Data Protection & Security
  • Encrypted connections (HTTPS) across all pages.
  • Secure, access-controlled database storage through our backend provider.
  • Passwords are hashed — never stored in plain text.
  • Access to user data is restricted to authorised personnel only.

No method of transmission over the internet is 100% secure. We retain your personal data only as long as necessary. If you request deletion, we will remove your data within a reasonable timeframe.

6. Your Rights
  • Access — Request a copy of the personal data we hold about you.
  • Correction — Request that inaccurate data be corrected.
  • Deletion — Request that we delete your personal data. Contact us directly or delete your account from settings.

How KamiStream Started

KamiStream was born from a simple frustration: anime fans deserved a better home on the internet. A place that wasn't cluttered, slow, or cold — somewhere that felt alive, creative, and built by people who actually love anime.

We started as a small project with one goal: make it easier for fans to find and watch anime, and give creators a place to show off their passion. Today, KamiStream combines a growing library of embedded anime content with a weekly creator challenge that puts real cash prizes in the hands of talented fan editors and animators.

Common Questions
How do I join the creator challenge?
Create a free account, go to the Challenge tab, and paste your video link. Submissions open weekly — check the timer on the homepage for the current deadline.
How do I request removal of a video?
For copyright concerns, email dmca@kamistream.tv with a full DMCA notice. See our for the full process.
A video isn't loading — what should I do?
Try switching servers using the server buttons in the player. If the issue persists, it may be the third-party source. You can report it using the form above.
How do I delete my account or data?
Email support@kamistream.tv with your username. We'll process the deletion within 7 business days.
